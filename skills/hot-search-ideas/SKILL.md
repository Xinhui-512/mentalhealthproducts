# 热搜产品创意分析技能

## 概述

本技能帮助用户从全网热搜榜单中筛选**心理健康、个人成长、人际关系、情绪管理、减压放松**相关的热点话题，提取产品创意灵感，生成结构化的产品创意报告。

## 内容筛选范围

**重点关注的热点话题类别**（与心理健康/个人成长/人际关系强相关）：

**数量要求**：每次筛选不少于**10条**相关话题

**详细程度要求**：每个话题必须包含详细的事件概要（50-150字），描述事件背景、最新动态或引发讨论的原因。不能只有标题和评分，读者应该能从报告中了解"发生了什么"。

**产品创意要求**：
- 简单轻量：优先小程序/轻应用，不需要复杂架构
- 蓝海市场：避开已经饱和的赛道（如泛心理咨询、通用日记本）
- 创新方向：寻找差异化切入点，解决细分场景痛点

| 类别 | 关键词/话题方向 |
|------|----------------|
| **职场压力** | 加班、裁员、打工人、996、职场PUA、晋升受阻、工资低、裸辞、35岁危机、试用期 |
| **生存状态** | 租房、买房困难、存款焦虑、月光族、还贷压力、生活成本高、"活不起" |
| **情感婚恋** | 恋爱、分手、离婚、相亲、催婚、单身、婆媳矛盾、夫妻关系、重男轻女、彩礼 |
| **家庭关系** | 亲子教育、鸡娃、厌学、叛逆期、原生家庭、父母沟通、养老、独居老人 |
| **心理健康** | 焦虑、抑郁、失眠、崩溃、孤独感、情绪失控、心理疾病、自杀倾向、心理咨询 |
| **社会现象** | 躺平、摆烂、佛系、内卷、空巢青年、剩女、相亲角、相亲软件 |
| **消费心理** | 冲动消费、报复性消费、透支、花呗、断舍离、极简主义、购物成瘾 |

**关联判断标准**：
- 话题主角是"人"而非"政策/科技"
- 反映普通人（尤其是年轻人）的生存处境和情感需求
- 引发广泛共鸣和讨论，评论区呈现焦虑/压力共鸣
- 体现某种社会情绪（孤独、焦虑、迷茫、愤怒）

**排除的类别**：
- 纯政治新闻、军事、国际冲突
- 体育赛事（除非与心理健康相关）
- 科技/数码新品发布（除非与消费心理相关）
- 财经股市波动（除非与普通人生存焦虑相关）
- 美食探店、娱乐八卦（除非与情绪减压/社交相关）

## 工作流程

### 第一阶段：抓取热搜榜单

1. **获取热搜数据**：使用用户提供的API接口抓取热搜榜单
   - 调用用户指定的API端点获取热搜列表
   - 解析返回的JSON数据，提取热搜话题、排名、热度值等关键信息
   - 支持微博、抖音、知乎等多平台热搜

2. **数据整理**：将热搜数据整理成标准化格式
   ```json
   {
     "platform": "微博",
     "hot_list": [
       {"rank": 1, "topic": "话题名称", "hot_value": 1234567, "date": "2026-04-11", "source": "微博热搜"},
       ...
     ],
     "fetch_time": "2024-01-01 12:00:00"
   }
   ```

   **每个热搜条目必须包含**：
   - `rank`：热搜排名
   - `topic`：话题名称
   - `hot_value`：热度值
   - `date`：该热点事件的发生/曝光日期（格式：YYYY-MM-DD）
   - `source`：新闻来源（如：央视新闻、微博热搜、知乎日报等）

### 第二阶段：内容筛选

从抓取的热搜中筛选符合**内容筛选范围**的话题：

1. **关键词匹配**：初步筛选包含以下关键词的话题
   - 心理类：焦虑、抑郁、失眠、心理健康、心理疏导、心理医生、情绪、压力、崩溃、治愈等
   - 成长类：学习、提升、成长、职场、晋升、加薪、辞职、找工作、副业、技能等
   - 关系类：亲子、恋爱、婚姻、夫妻、婆媳、友情、社交、人际、沟通、矛盾等
   - 情绪类：情绪、发脾气、愤怒、悲伤、孤独、快乐、幸福、正念、冥想、心态等
   - 放松类：旅游、娱乐、休闲、度假、游戏、追剧、美食、解压、躺平等

2. **AI二次筛选**：对初步筛选结果进行语义分析，确认是否与五大类相关
   - 如果热搜表面不相关但评论区/延伸内容与五大类相关，也可纳入
   - 如果热搜相关但过于专业化或小众，可标注后纳入

3. **输出筛选结果**：从40条热搜中筛选出5-10条最相关的话题进行分析
   - 如果相关话题不足10条，选择最相关的补充
   - 如果超过10条，按相关度和热度综合排序

4. **生成详细的事件概要**：对每个话题，根据AI知识生成详细的背景/动态描述（50-150字）
   - 让读者能快速了解"发生了什么"
   - 描述事件起因、现状、或引发讨论的原因
   - 如果话题较为知名，可以描述得更详细

### 第三阶段：热点详情搜索

对每个热搜话题执行以下操作：

1. **使用WebSearch工具**搜索该话题的最新新闻、背景信息
2. **搜索关键词格式**：`"{热搜话题}" 最新消息` 或 `"{热搜话题}" 事件详情`
3. **收集信息包括**：
   - 事件起因和经过
   - 当前状态和舆论反应
   - 相关人物和机构态度
   - 后续发展趋势

### 第四阶段：AI产品创意分析

对每个热点进行产品创意分析，评分标准：

- **有趣度 (80分)**：创意的趣味性、话题性、传播潜力
- **有用度 (20分)**：实际解决用户需求的程度

**总分 = 有趣度 × 0.8 + 有用度 × 0.2**

#### 产品创意输出格式

```json
{
  "topic": "热搜话题名称",
  "event_summary": "事件脉络摘要（50-150字，描述事件背景、最新动态或引发讨论的原因）",
  "product_ideas": [
    {
      "name": "产品名称",
      "core_functions": ["核心功能1", "核心功能2"],
      "target_users": "目标用户描述",
      "fun_score": 75,
      "utility_score": 18,
      "total_score": 77.4
    }
  ]
}
```

**事件概要（event_summary）要求**：
- 字数：50-150字
- 内容：事件背景、最新动态、或引发广泛讨论的原因
- 让读者能够快速了解"这个热搜在说什么"
- 如果话题较为知名或复杂，可以描述得更详细

### 第五阶段：生成HTML报告

生成一个列表式的HTML分析报告：

**报告结构**：
1. 标题区：报告标题、生成时间
2. 统计概览：热搜数量、分析话题数、优秀/良好评级分布
3. 热点列表（按总分降序排列）：
   - 热点话题和排名
   - **日期**：该热点的发生/曝光日期
   - **来源**：新闻出处（如：央视新闻、微博热搜、知乎等）
   - **事件概要**：详细的背景/动态描述（50-150字），让读者了解"发生了什么"
   - 产品创意列表
   - 综合评分及等级标识

**每个话题必须包含详细的事件概要**，不能只有标题和评分。

**评分等级**：
- **优秀 (≥80分)**：金色高亮标记
- **良好 (60-80分)**：银色/蓝色标记
- **一般 (<60分)**：普通显示

**HTML样式要求**：
- 响应式布局，适配桌面和移动端
- 使用清晰的视觉层次
- 评分用进度条或数字展示
- 优秀创意使用金色边框或背景突出
- 代码使用等宽字体展示

## 执行示例

用户输入：`分析今天的热搜，生成产品创意报告`

执行步骤：
1. 调用API获取热搜数据
2. 对每个热搜话题执行WebSearch
3. AI分析并生成产品创意
4. 生成HTML报告并保存

## 输出文件

生成的HTML报告保存为：`hot_search_report_{YYMMDD}.html`
- 日期格式：`YYMMDD`（如2026年4月11日 → `260411`）
- 完整文件名示例：`hot_search_report_260411.html`

## 依赖工具

- WebSearch：搜索热点新闻
- Bash：执行API调用和文件操作
- Write/Edit：生成HTML文件

## 注意事项

1. API调用失败时，提示用户检查网络或API配置
2. 搜索无结果的话题，跳过该话题或标注"信息不足"
3. 产品创意要有创意但要接地气，避免过于天马行空
4. HTML报告中的中文要使用UTF-8编码
